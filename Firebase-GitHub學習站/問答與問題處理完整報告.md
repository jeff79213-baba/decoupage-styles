## 一、開發完成後的問題回報

### 問題 1：頁面功能異常（07:08）

**使用者描述：**
> Git/GitHub 基礎 現在只有這個頁面有東西。模擬：第一次 Push 到 GitHub 按了沒反應。小測驗也沒反應。其他頁面都無法點擊。

**問題分析：**
JS 字串中有 3 處未跳脫的 ASCII 單引號，導致整個 `<script>` 區塊解析失敗，所有 JavaScript 功能全部失效。

**修復方式：**
1. 3 處致命 JS 語法錯誤 — 把 `'text'` 改成 `「text」`
2. quizGroup 代號不一致 — mod1 的 5 題統一使用 `group=1`
3. mod1 Q1、Q2 的 `data-answer` 值指向錯誤答案 — 修正為正確索引

## 二、關於程式碼與檔案的提問

### 問題 2：檔案名稱可以改嗎？

**使用者問：** 檔案的名稱可以改嗎？

**回答：** 可以，改成任何名字都行，只要副檔名保持 `.html` 就能正常運作。

---

### 問題 3：按 F12 能看到裡面的資料嗎？

**使用者問：** 幫我打開來按 F12 我可以看到裡面的資料嗎？

**回答：** 可以。F12 開啟瀏覽器開發者工具（DevTools），可以看到：
- Elements（元素）— 完整 HTML 結構 + CSS 樣式
- Sources（原始碼）— `<script>` 裡的所有 JavaScript 程式碼
- Console（主控台）— JS 錯誤訊息

---

### 問題 4：檔案如何在別的平台使用？

**使用者問：** 這個檔案我要怎麼樣可以在別的平台用，比如說把它傳到 Line 或 Mail 然後用別的電腦打開？

**回答：**
| 平台 | 操作 |
|------|------|
| Line | 把 `index.html` 拖進聊天視窗當檔案傳送 |
| Email | 附件夾帶 `index.html` 寄出 |
| USB | 複製檔案過去，雙擊打開 |
| 雲端硬碟 | 上傳後下載，雙擊打開 |

對方收到後直接雙擊 `index.html`，就會用瀏覽器打開，不需要安裝任何東西。

## 三、概念理解的提問

### 問題 5：HTML 檔案的概念類比

**使用者問：** 是不是就像是不同的 EXCEL、WORD，資料都在整個檔案裡面，只是它的名稱是 HTML？

**回答：** 完全正確。
| 副檔名 | 用什麼打開 | 裡面裝什麼 |
|--------|-----------|-----------|
| `.xlsx` | Excel | 試算表資料 |
| `.docx` | Word | 文字和排版 |
| `.html` | 瀏覽器 | 網頁的內容和程式碼 |

所有的文字、顏色、動畫、問答邏輯全部封在這一個檔案裡，搬到任何一台電腦都能直接跑。

---

### 問題 6：這種模式的專業名稱

**使用者問：** 像這樣的模式用專業一點的說法是不是就叫做「前端靜態」？

**回答：** 對，專業說法是**「前端靜態網頁」（Static Frontend）**。
- **前端**：程式跑在使用者的瀏覽器上（不是伺服器上）
- **靜態**：內容寫死在檔案裡，不會根據使用者或時間自動改變

與之相反的是「動態網頁」，例如登入 Facebook 後每個人看到的頁面不一樣。

---

### 問題 7：懂程式的人按 F12 能修改變更嗎？

**使用者問：** 懂程式的人按 F12 都可以修改變更嗎？

**回答：**
| 情境 | 能改？ | 有影響？ |
|------|:------:|:--------:|
| F12 改畫面 | 能 | 只影響自己，重新整理就恢復原狀 |
| 拿到檔案用編輯器改 | 能 | 永久改變那個檔案 |
| 沒拿到檔案 | 不能改 | — |

F12 改的東西只存在於自己的瀏覽器，按 F5 重新整理就恢復原狀，其他人的電腦完全不受影響。真正能永久修改的是拿到 `index.html` 檔案本身，用文字編輯器打開直接改程式碼。

## 四、Line 傳送後功能失效的問題

### 問題 8：Line 預覽模式下互動功能失效（08:03 後）

**使用者描述：**
> 我丟到 Line 上，會看到是一個按鈕，有 .ZIP，然後我點擊，它需要下載之後，預覽內容的方式打開來，有看到頁面了可是跟昨天一樣，我點擊任何按鈕，它不會有反應或是跳出該要有的答案。

**問題分析：**
Line 的「預覽內容」功能使用應用程式內建的簡易瀏覽器，為了安全起見會封鎖 JavaScript 的執行。學習站的所有互動功能全部依賴 JavaScript，因此在預覽模式下全部失效。

**解決方式：**
不要使用 Line 的預覽功能，改用：
1. 在 Line 聊天視窗中長按或右鍵點擊檔案
2. 選擇「下載」或「另存新檔」
3. 到下載資料夾中找到 `index.html`
4. 右鍵 → 開啟方式 → 選擇 Chrome 或 Edge

| 打開方式 | JavaScript | 互動功能 |
|----------|:----------:|:--------:|
| Line 預覽 | ❌ 被封鎖 | 全部失效 |
| 下載後用 Chrome/Edge/Firefox 開啟 | ✅ 正常執行 | 全部正常 |
