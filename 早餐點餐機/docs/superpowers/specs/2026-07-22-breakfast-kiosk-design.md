# 早餐點餐機 — 設計規格書（完整版）

## 1. 專案概述

建立一個**平板全螢幕 kiosk 風格**的早餐點餐系統。採用「共用程式碼模板 + 獨立專案資料夾」架構，每個店家有自己的 JSON 菜單資料與客製化設定。

### 目標設備
- 平板橫向全螢幕（1024x768 以上）
- 觸控操作為主

### 核心特色
- 左側分類 + 品項選單，右側購物車，底部配料區
- 每個分類有獨立配料選項（配料庫可跨分類共用）
- 品項可多選，配料只作用於「亮燈品項」
- 購物車同品項同配料自動合併顯示數量
- 支援 內用/外帶 選擇
- 雙主題切換（Dark / Sage），後台設定
- 支援 Thermal Printer 擴充（目前顯示「功能尚未開啟」）
- QR Code 全店共用，多桌同時掃描不衝突
- Firebase 雲端同步，使用量追蹤

## 2. Firebase 架構

### 專案結構
- **一個共用 Firebase 專案**，所有店家都在裡面
- 資料用 `shops/{店家ID}/` 區分
- 每個店家有獨立的菜單、訂單、使用量紀錄

### Firestore 資料結構

```
shops/
  ├── burger-shop-001/
  │     ├── menu (文件)          ← 菜單資料
  │     ├── stats (文件)         ← 使用量追蹤
  │     └── orders/ (子集合)     ← 訂單紀錄
  ├── egg-pancake-shop-002/
  │     ├── menu (文件)
  │     ├── stats (文件)
  │     └── orders/ (子集合)
  └── ...
```

### stats 文件內容

```json
{
  "shopName": "好吃早餐店",
  "dailyStats": {
    "2026-07-22": { "reads": 120, "writes": 45 },
    "2026-07-23": { "reads": 98, "writes": 38 }
  },
  "totalStats": {
    "reads": 15000,
    "writes": 5800
  },
  "retentionDays": 365
}
```

- `dailyStats`：每天使用量，超過保留天數自動清除（預設 365 天）
- `totalStats`：累計總量（不清除）
- 客人掃 QR Code 載入菜單 → stats.reads +1
- 客人確認訂單 → stats.writes +1
- 後台改菜單 → stats.writes +1

### QR Code 規則
- 全店一個 QR Code，指向 `xxx.com/?shop={店家ID}`
- 多桌同時掃描 → 各自獨立的點餐畫面（localStorage 獨立）
- 菜單從 Firestore 即時載入，改完菜單後下一位客人掃到的就是最新版
- QR Code 內容是固定網址，不會因菜單變更而改變

### 免費額度與容量

| 項目 | 100 店/天用量 | 免費額度 | 夠不夠？ |
|------|-------------|---------|---------|
| 讀取次數 | ~10,000 | 50,000 | ✓ |
| 寫入次數 | ~5,000 | 20,000 | ✓ |
| 儲存空間 | ~10 MB | 1 GB | ✓ |
| Bandwidth | ~300 MB/月 | 10 GB/月 | ✓ |

100 家店以內免費版綽綽有餘。超出後即用即付，500 家店月費約 NT$13。

### 使用量追蹤
- 每店的 stats 文件記錄 dailyStats（每日 reads/writes）
- 保留期限：365 天（可調整）
- 超過保留天數的 dailyStats 自動清除
- 總管理後台可查看所有店家使用量排行，判斷哪一家需要獨立專案

### 規則
- `addonLibrary`：全局配料庫，所有配料集中管理

- `items[].enabled`：false = 下架（不顯示但保留）
- `items[].sortOrder` / `categories[].sortOrder`：排序
- 資料存在 Firestore，後台改完即時同步到所有裝置

### 內用/外帶按鈕
- 位置：右上角，兩個並排按鈕
- 預設無選取狀態（兩按鈕都是一般樣式）
- 點擊後該按鈕變為選中樣式（primary 色）
- 選取後記住狀態，加入購物車時帶入
- 若未選取，按「確認訂單」時彈窗詢問

### 品項卡片三種狀態

| 狀態 | 外觀 | 功能 |
|------|------|------|
| **未選** | 正常灰底 | 點擊後變為「亮燈」 |
| **亮燈（Active）** | 深色反白、框加粗 | 可加配料、配料作用於此品項 |
| **反白 + Xn（Selected）** | 淺色反白、顯示數量 | 已選，不影響配料區 |

- 同一品項可被選多次（X2、X3...），但只有**最後一個**是「亮燈」
- 只有「亮燈」的品項可以接受配料
- 點擊「反白+Xn」的品項 → 變為亮燈，前一個亮燈品項變為反白

### 取消按鈕
- 亮燈品項旁有取消按鈕
- 按取消 → 該品項歸零（變回未選）

## 5. 點餐流程（完整步驟）

### 5.1 開啟點餐機
- 載入 `menu.json`
- 顯示店名、副標題
- 內用/外帶按鈕顯示在右上角，預設未選取

### 5.2 選擇分類
- 點擊左側分類按鈕
- 品項列表更新為該分類的品項
- 底部配料區更新為該分類的配料

### 5.3 選擇品項
1. 點擊品項 → 變為「亮燈」
2. 可同時選多個品項（各有自己的亮燈/反白狀態）

### 5.4 選擇配料
1. 僅「亮燈」的品項可加配料
2. 點擊配料按鈕 → 亮燈品項卡片上新增配料標籤
3. 再點同一配料 → 取消該配料（toggle）

### 5.5 加入購物車
1. 按「加入購物車」按鈕
2. **所有有 X1 的品項（含配料）一次性加入購物車**
3. 畫面上所有品項狀態歸零
4. 購物車自動更新

### 5.6 購物車合併邏輯
- 同品項 + 同配料 → 合併顯示，數量 +1
- 同品項 + 不同配料 → 分開顯示

```
豬肉漢堡              X2    $130    （2 個無配料）
豬肉漢堡 +蛋          X1    $75     （1 個加蛋）
牛肉漢堡 +蛋 +起司    X1    $100    （1 個加蛋加起司）
```

### 5.7 購物車編輯
- 點擊購物車內的品項 → 展開明細
- 可修改配料（增/刪）
- 可調整數量（+/-）
- 可刪除整個品項

### 5.8 切換分類
- 切換分類**不清除**前分類的選項狀態
- 可隨時切回前分類繼續新增品項
- 購物車是獨立的，不受分類切換影響

- **Sage**：森林綠 #0E3E37 + 芥末金 #B58D3D（有機自然感）
- 切換在後台設定，消費者端不顯示切換按鈕
- 切換時所有頁面同步更新

### 共用設計語言
- 字體：Inter（Dark）/ Montserrat（Sage）
- 大圓角：16px-24px
- 膠囊型按鈕
- 扁平化設計，無陰影

## 9. 技術規格

- **前端**：純 HTML + CSS + JS（無框架依賴）
- **雲端資料庫**：Firebase Firestore（菜單、訂單、使用量）
- **本機儲存**：localStorage（購物車暫存）
- **部署**：Firebase Hosting
- **QR Code**：全店一張，指向 `xxx.com/?shop={店家ID}`
- **響應式**：CSS Grid / Flexbox，支援橫向全螢幕
- **主題系統**：CSS 變數（`--bg`, `--surface`, `--primary` 等）
