# 點餐系統庫存整合研究報告

> 建立日期：2026-06-20  
> 討論範疇：芭拉咘咘麵包車（現有 HTML 點餐系統）

---

## 1. 需求摘要

當客人在網頁點餐、按下「結帳送出」後，系統需要自動扣減庫存量。

### 流程

```
客人點餐 → 加入購物車（先不檢查庫存）
            ↓
     客人到購物車按「結帳」
            ↓
     從 Firebase 抓取最新庫存
            ↓
     逐一檢查每個品項庫存是否足夠
            ↓
     ┌─── 全部足夠 ─── 扣庫存 → 寫入 Firebase → 產生訂單 → 清空購物車
     └─── 有不夠的 ─── 彈出警告：「XXX 只剩 X 個，您點了 Y 個」
                           停留在購物車頁面，不讓結帳
```

---

## 2. 選擇記錄

| 項目 | 選擇結果 |
|------|---------|
| 庫存存放位置 | **Firebase Firestore（雲端）** — 跨裝置即時同步 |
| 庫存層級 | **每個品項總庫存** — 不論哪個價位都扣同一份庫存 |
| 庫存為 0 的行為 | **可加入購物車，但結帳時檢查並擋住** |

## 3. 資料結構設計

### Firestore 結構

```
stock (collection)
  └── 芭拉咘咘麵包車 (document id = 店家名稱)
        ├── updatedAt: Timestamp
        └── items: {
              "菠蘿麵包": { stock: 20 },
              "蔥花麵包": { stock: 15 },
              "紅豆麵包": { stock: 10 },
              "蜂蜜蛋糕": { stock: 5 },
              "白吐司":   { stock: 8 },
              ...
            }
```

- **key** 是品項名稱（字串）
- **stock** 是整數庫存量
- 同一個品項不管選哪個價位（單個 / 三個），都扣同一個庫存

### JavaScript 全域變數（新增在 HTML 的 `<script>` 中）

```js
let STOCK = {};   // 例如: { "菠蘿麵包": 20, "蔥花麵包": 15 }
```


### 步驟 3：新增全域變數

```js
let STOCK = {};
```

### 步驟 4：新增庫存讀寫函數

```js
async function loadStockFromFirebase() {
  try {
    const doc = await db.collection("stock").doc(SHOP_ID).get();
    if (doc.exists) {
      STOCK = doc.data().items || {};
    } else {
      STOCK = {};
    }
  } catch (e) {
    console.error("讀取庫存失敗", e);
    showToast("無法讀取庫存資料");
  }
}

async function saveStockToFirebase() {
  try {
    await db.collection("stock").doc(SHOP_ID).set({
      items: STOCK,
      updatedAt: firebase.firestore.FieldValue.serverTimestamp()
    });
  } catch (e) {
    console.error("儲存庫存失敗", e);
    showToast("庫存更新失敗，請重試");
  }
}
```

### 步驟 5：修改 `init()` 函數 — 載入時抓庫存

在 `loadData()` 之後加上：

```js
await loadStockFromFirebase();
```

因為 `init()` 原本不是 async，需要改成：

```js
async function init() {
  loadData();
  await loadStockFromFirebase();
  // ... 其餘初始化程式碼

```

### 步驟 6：修改 `renderMenu()` — 顯示庫存

在每個品項卡片中，品項名稱旁邊加上庫存資訊。  
在 `item.name` 顯示之後加：

### 步驟 7：修改 `checkout()` — 結帳檢查並扣庫存

**新流程取代原本單純的 `checkout()`：**

```js
async function checkout() {
  if (!CART.length) return;

  // 1. 重新抓取最新庫存
  await loadStockFromFirebase();

  // 2. 檢查每個品項庫存是否足夠
  const insufficient = [];
  for (const c of CART) {
    const itemName = c.itemName;
    const currentStock = STOCK[itemName]?.stock ?? Infinity;
    if (currentStock !== Infinity && c.qty > currentStock) {
      insufficient.push({ name: itemName, stock: currentStock, ordered: c.qty });
    }
  }

  // 3. 如果有不足，擋住結帳
  if (insufficient.length > 0) {
    const msg = insufficient.map(
      i => `${i.name}：只剩 ${i.stock} 個，你點了 ${i.ordered} 個`
    ).join("\n");
    showToast("庫存不足！\n" + msg);
    return;
  }

  // 4. 庫存足夠 → 扣庫存（先扣再寫訂單，防止同時多人下單）
  for (const c of CART) {
    const itemName = c.itemName;
    if (STOCK[itemName]) {
      STOCK[itemName].stock -= c.qty;
    }
  }

  // 5. 寫入 Firebase
  await saveStockToFirebase();

  // 6. 產生訂單（原本的邏輯）
  const now = new Date();
  const order = {
    id: Date.now(),
    date: `${now.getFullYear()}/${String(now.getMonth()+1).padStart(2,"0")}/${String(now.getDate()).padStart(2,"0")}`,
    time: `${String(now.getHours()).padStart(2,"0")}:${String(now.getMinutes()).padStart(2,"0")}`,
    items: CART.map(c => ({
      name: c.itemName,
      priceLabel: c.priceLabel,
      unitPrice: c.unitPrice,
      quantity: c.qty,
      subtotal: c.unitPrice * c.qty
    })),
    total: CART.reduce((s, c) => s + c.unitPrice * c.qty, 0)
  };

  ORDERS.unshift(order);
  saveOrders();
  CART = [];
  saveCart();
  renderCart();
  updateCartBadge();
  showToast("結帳成功！");
}
```

### 步驟 8：修改 `cancelOrder()` — 取消訂單加回庫存

```js
async function cancelOrder(index) {
  showConfirm("確定要取消此訂單嗎？", async () => {
    const order = ORDERS[index];
    // 加回庫存
    for (const item of order.items) {
      if (STOCK[item.name]) {
        STOCK[item.name].stock += item.quantity;
      }
    }
    await saveStockToFirebase();

    ORDERS.splice(index, 1);
    saveOrders();
    renderOrders();
    showToast("訂單已取消，庫存已加回");
  });
}
```

## 6. Firebase 費用評估

### 免費額度（Spark 方案）

| 項目 | 免費額度 | 一家麵包店估算用量 |
|------|---------|------------------|
| Firestore 讀取 | 50,000 次/天 | 頁面載入 × 5 + 結帳檢查 × 1 = 約 6 次/人 |
| Firestore 寫入 | 20,000 次/天 | 每筆訂單寫入 1 次 |
| 儲存空間 | 1 GB | 庫存+訂單資料遠低於 1 MB |

**一家店每天 500 張訂單、500 人次瀏覽 → 還在免費額度內。**

### 超過後的費用（Blaze 方案）

| 項目 | 超過後單價 |
|------|-----------|
| 讀取 | $0.18 美金 / 10 萬次 |
| 寫入 | $0.18 美金 / 10 萬次 |
| 儲存 | $0.18 美金 / GB / 月 |

**最低付費：每月 $0.18 美金起（約台幣 6 元）**

### 模擬情境

```
每天 3,000 張訂單（生意非常好）：
  寫入：30,000 次/天 → 超過 10,000 次/天
  每月額外：10,000 × 30 = 300,000 次超額
  費用：300,000 ÷ 100,000 × $0.18 = $0.54 美金（約 17 台幣）

加上讀取額外費用：約 $1 美金
每月總計：約 $1.5 美金（約 50 台幣）
```

### 設定費用上限

在 Google Cloud Console → 帳單 → 預算與通知，設定每月預算上限（例如 $5 美金），超過自動通知或停用。

## 8. 多人即時連動說明

```
客人（瀏覽器） ←→ Firebase Firestore ←→ 老闆（瀏覽器）
  下單                  雲端資料庫           看新訂單、管庫存
```

### 即時同步原理

Firestore 支援 `onSnapshot`（即時監聽），不需要重新整理網頁：

```js
// 老闆端監聽庫存變化
db.collection("stock").doc(SHOP_ID).onSnapshot((doc) => {
  if (doc.exists) {
    STOCK = doc.data().items || {};
    if (document.getElementById("view-menu").classList.contains("active")) {
      renderMenu();  // 庫存更新時自動刷新菜單畫面
    }
  }
});
```

### 多客人同時下單的衝突處理

使用 Firestore **transaction**（交易）來確保同時扣庫存不會出錯：

```js
async function safeDecreaseStock(itemName, qty) {
  try {
    await db.runTransaction(async (transaction) => {
      const ref = db.collection("stock").doc(SHOP_ID);
      const doc = await transaction.get(ref);
      const items = doc.data().items;
      if (!items[itemName] || items[itemName].stock < qty) {
        throw new Error(`${itemName} 庫存不足`);
      }
      items[itemName].stock -= qty;
      transaction.update(ref, { items });
    });
    return true;
  } catch (e) {
    showToast(e.message);
    return false;
  }
}
```

## 9. 實作注意事項

1. **備份原始檔案** — 修改前先複製一份 `index.html` 備份
2. **async/await 傳染** — 所有涉及 Firebase 的函數都要加上 `async`，呼叫端也要 `await`
3. **測試順序**：
   - 先測：`loadStockFromFirebase()` 能否正確讀取
   - 再測：管理後台能否設定庫存
   - 再測：前台能否看到庫存數字
   - 再測：結帳能否正確扣庫存
   - 最後測：庫存不足時是否擋住
4. **取消訂單** — 記得加回庫存，否則庫存會愈賣愈少
5. **首次使用者** — 首次載入時如果 Firestore 還沒有該店家的文件，`STOCK` 會是 `{}`，顯示時要處理 undefined 情況（顯示「--」或「未知」）

## 10. 未來擴充建議

- **低庫存通知** — 庫存低於某個數量（如 5 個）時，在管理後台顯示紅字警告
- **訂單狀態管理** — 新增「準備中、已完成、已取餐」等狀態
- **庫存歷史記錄** — 記錄每次庫存異動（進貨、出貨、退貨）
- **多店家支援** — 一個 Firebase 專案同時管理多家店（用不同的 document id）
- **匯入/匯出庫存** — 批次從 Excel 匯入庫存資料
- **與 Firebase Authentication 整合** — 老闆用 Email 登入管理後台，取代原本的密碼保護
