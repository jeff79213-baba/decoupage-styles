從零做一款像「殭屍大逃殺」的遊戲 — 步驟與需要動腦的地方
=============================================================

這份報告以「殭屍大逃殺」為參考，拆解做一款 3D 網頁遊戲的完整流程，
以及每個環節中「哪些事情必須由你（設計者）親手決定」。

先說結論：AI（Claude）能幫你寫程式碼、解決技術問題，
但遊戲的「骨幹」— 玩法、平衡、數值 — 全部要靠你自己想。

====================================================================
一、前置準備（技術門檻）
====================================================================

  □ 學基礎 HTML / CSS / JavaScript（至少看得懂程式在幹嘛）
  □ 會用 VS Code 編輯器
  □ 會開終端機執行指令（git、node 等）
  □ 有 GitHub 帳號（免費）

  🤔 需要動腦的地方：
     幾乎沒有。這些都是工具操作，學就會。

====================================================================
二、決定遊戲玩法（最重要的一步）
====================================================================

  你要先想清楚「這到底是什麼遊戲」，具體包含：

  □ 遊戲類型：射擊？動作？益智？回合制？
  □ 勝利條件：打幾關？打幾隻王？存活多久？
  □ 失敗條件：血量歸零？
  □ 操作方式：鍵盤滑鼠？觸控？搖桿？
  □ 遊戲循環：打怪→撿經驗→升級→變強→打更強的怪

  範例（殭屍大逃殺）：
    - 類型：倖存者類 roguelite（參考 Vampire Survivors）
    - 勝利：擊敗 7 隻王
    - 失敗：血量歸零
    - 操作：走位 + 自動攻擊
    - 循環：殺殭屍 → 撿經驗寶石 → 升等選升級 → 變強 → 更難的殭屍

  🤔 需要動腦的地方：★★★★★（最難）
     • 你的遊戲「好玩在哪」？這是沒辦法抄的。
     • 你必須找到一個讓玩家願意一直玩的循環。
     • 建議先玩類似遊戲（Vampire Survivors、Broforce 等），
       思考它們為什麼好玩。

====================================================================
三、設計美術風格與取得素材
====================================================================

  有幾種路線：

  路線 A：免費 3D 模型包（推薦）
    - 去 Quaternius 等網站下載 CC0 免費包
    - 優點：品質好、有骨架動畫、不用錢
    - 缺點：風格不能自己控制、選擇有限

  路線 B：AI 生成圖片
    - 用 Midjourney / Stable Diffusion 生圖
    - 優點：風格自訂
    - 缺點：要花時間生、要手動去背/切圖、不能直接變成可動 3D

  路線 C：純文字/Emoji/幾何形狀
    - 像「數學對戰」那樣全部用 Emoji 和 CSS
    - 優點：最快、零美術成本
    - 缺點：看起來就是很陽春

  🤔 需要動腦的地方：★★★☆☆
     • 選擇哪條路線 → 你的資源和時間決定
     • 如果是路線 A → 找素材、挑選哪些模型符合遊戲需求
     • 如果是路線 B → 生圖 prompt、後製
     • 最關鍵的決定：你的遊戲需要什麼樣的視覺風格？
       歡樂？暗黑？卡通？寫實？

====================================================================
四、設計升級與數值（遊戲平衡）
====================================================================

  這是「殭屍大逃殺」最需要腦力的部分之一。
  每一種升級你都要手動設計：

  □ 升級名稱：攻擊力、攻速、多重彈、護甲……
  □ 效果描述：子彈 +1、傷害 +20%……
  □ 數值曲線：每級加多少？最多幾級？
  □ 平衡性：會不會某個升級太強或太廢？

  範例，開發者寫在 upgrades.ts 裡的是這樣：
    { id: 'damage', maxLevel: 8, apply: (s) => s.damage += 1 }
    { id: 'firerate', maxLevel: 8, apply: (s) => s.fireInterval *= 0.88 }

  25 種升級，每一種都要你自己想。

  🤔 需要動腦的地方：★★★★★（跟玩法並列最難）
     • 每一種升級的數值都要你決定
     • 你必須一直問自己：「這樣會太強嗎？太弱嗎？」
     • 一定要找人試玩 → 根據回饋調整數值（這叫「平衡 Iteration」）
     • 數字設定沒有標準答案，只能靠測試和經驗

====================================================================
五、設計敵人與王（行為模式）
====================================================================

  每一種敵人都要設計：

  □ 外觀：用哪個模型
  □ 血量：多少 HP
  □ 速度：移動多快
  □ 攻擊模式：衝撞？遠程？範圍？
  □ 特殊行為：蓄力、召喚小兵、彈幕……

  殭屍大逃殺有 6 種小兵 + 7 隻王，
  每隻王都有獨特的招式（衝撞、彈幕、毒池、震波…）

  🤔 需要動腦的地方：★★★★☆
     • 每種敵人的行為都要自己設計
     • 王的打法要「有規律可閃避」，不能單純硬打
     • 7 隻王的難度曲線要循序漸進
     • 每隻王要讓玩家死過幾次才能找到打法

====================================================================
六、選擇技術棧與寫程式
====================================================================

  這部分是 AI 最擅長的。你只需要決定用什麼工具：

  □ 遊戲引擎：
    - Babylon.js（3D 網頁，殭屍大逃殺用的）
    - Three.js（另一套 3D 網頁引擎）
    - Phaser（2D 網頁遊戲）
    - Unity / Godot（要下載安裝，不是純網頁）

  □ 前端框架（可選）：
    - Vue 3（殭屍大逃殺用的）
    - React
    - 什麼都不用，單頁 HTML

  □ 後端（如果有需要）：
    - Cloudflare D1（殭屍大逃殺用的，排行榜用）
    - Firebase Realtime Database（數學對戰用的，即時同步）
    - 什麼都不用（純單機）

  □ 部署：
    - GitHub Pages（免費）
    - Cloudflare Pages（免費，殭屍大逃殺用的）
    - Vercel / Netlify

  決定好之後，把規格告訴 Claude，它會幫你寫程式碼。

  🤔 需要動腦的地方：★★☆☆☆
     • 選擇用哪套引擎（但 Claude 可以幫你分析優缺點）
     • 程式的邏輯架構（但 Claude 會幫你規劃）
     • 純寫程式的部分 → AI 幾乎都能處理

====================================================================
七、測試與調整（Iteration）
====================================================================

  遊戲做完第一版之後：

  □ 自己玩一遍 → 找出 bug、平衡問題
  □ 找朋友試玩 → 觀察他們的反應
  □ 根據回饋調整數值
  □ 加新功能
  □ 重複以上無限循環

  🤔 需要動腦的地方：★★★☆☆
     • 判斷哪些回饋是對的、哪些只是個人偏好
     • 決定「改哪裡」比「怎麼改」難

====================================================================
八、部署上線
====================================================================

  把遊戲放到網路上讓別人玩：

  □ 買網域（可選，免費子網域也行）
  □ 設定自動部署（Git push 就自動更新）
  □ 分享網址給朋友

  🤔 需要動腦的地方：★☆☆☆☆
     照步驟做就好，幾乎不需要思考。

====================================================================
九、總結 — 哪個環節最花腦力？
====================================================================

  腦力需求排行：

  1. ★★★★★ 玩法設計 — 遊戲到底好不好玩
  2. ★★★★★ 升級與數值平衡 — 太強太弱都要調
  3. ★★★★☆ 敵人與王的行為設計 — 要讓玩家覺得「有挑戰但能過」
  4. ★★★☆☆ 美術風格選擇 — 資源有限的情況下做取捨
  5. ★★☆☆☆ 技術工具選擇 — Claude 可以幫你分析
  6. ★☆☆☆☆ 部署上線 — 照做即可

  實際開發建議：

  不要一次想全部做完。先做「最小可玩版本」：
    → 1 個角色 + 1 種攻擊 + 1 種敵人 + 1 種升級
    → 自己玩 → 改 → 加功能 → 無限循環

  這樣你才能在「花太多時間」之前就確認：
  「這個遊戲基底是好玩的嗎？」

====================================================================
補充：哪些事情 AI 做得好 vs 做不好
====================================================================

  AI（Claude）擅長：
  ✅ 寫程式碼、修 bug
  ✅ 建議技術架構
  ✅ 解釋程式原理
  ✅ 幫你重構、優化

  AI 不擅長（還是要你自己來）：
  ❌ 決定遊戲好不好玩
  ❌ 設計數值平衡（它會給你建議，但對不對要靠測試）
  ❌ 知道你要做什麼樣的遊戲（你要先想清楚才能告訴它）
  ❌ 判斷美術風格（它會建議，但最終決定權在你）
